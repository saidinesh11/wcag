UX Accessibility Audit

MakeMyTrip Accessibility Audit

A WCAG 2.2 AA accessibility review of the MakeMyTrip flight booking experience, focused on inclusive design, keyboard usability, screen reader support, visual accessibility, and form usability.

Project Overview

Designing for a more inclusive booking experience

Role

UX Designer / Accessibility Auditor

MakeMyTrip – Flight Booking Flow

Standards

WCAG 2.2 AA, Section 508

Windows 11, Chrome Browser

Audit Process

How the evaluation was conducted

Tested focus order, navigation, dropdown behavior, and interactive controls.

Key Findings

Accessibility findings with image evidence

High WCAG 1.1.1 / 4.1.2

01. Social media icons lack accessible names

UX Observation: Social media icons rely only on visual recognition.

User Impact: Screen reader users may hear only “link” without context.

Recommendation: Add descriptive labels such as “Follow us on Facebook.”

High WCAG 1.4.3

02. Helper text has insufficient contrast

UX Observation: Helper text appears in light gray on a white background.

User Impact: Low-vision users may struggle to read booking guidance.

Recommendation: Increase contrast to at least 4.5:1.

Medium WCAG 1.4.10

03. Flight form does not reflow properly

UX Observation: Higher zoom levels require horizontal scrolling.

User Impact: Zoom users must scroll sideways to access form fields.

Recommendation: Stack form fields vertically on small screens.

High WCAG 2.1.1

04. Some controls are not fully keyboard accessible

UX Observation: Key booking controls need complete keyboard support.

User Impact: Keyboard-only users may struggle to complete actions.

Recommendation: Support Tab, Enter, Space, and Arrow keys.

Medium WCAG 2.4.1

05. No skip link for repeated navigation

UX Observation: No visible-on-focus skip link is provided.

User Impact: Keyboard users must tab through repeated navigation.

Recommendation: Add “Skip to main content.”

High WCAG 2.4.7

06. Focus states are difficult to identify

UX Observation: Interactive elements lack clear focus indicators.

User Impact: Keyboard users may lose track of their position.

Recommendation: Use a strong high-contrast focus ring.

Medium WCAG 2.2.2

07. Auto-rotating carousel lacks user control

UX Observation: Promotional content rotates automatically.

User Impact: Users may lose content before reading it.

Recommendation: Add pause, stop, previous, and next controls.

High WCAG 1.3.1

08. Form labels and relationships need clarity

UX Observation: Form labels need clear programmatic relationships.

User Impact: Assistive technology users may not understand each field.

Recommendation: Associate every input with a visible label.

Outcome

Final Reflection

This audit showed how accessibility improvements directly improve usability for all users. Addressing keyboard navigation, screen reader support, visual readability, responsive behavior, and form clarity can make the booking experience more inclusive and easier to use.
