UX Accessibility Audit

MakeMyTrip Accessibility Audit

A WCAG 2.2 AA accessibility review of the MakeMyTrip flight booking experience, focused on inclusive design, keyboard usability, screen reader support, visual accessibility, and form usability.

View Findings Audit Process

Project Overview

Designing for a more inclusive booking experience

Role

UX Designer / Accessibility Auditor

Website

MakeMyTrip – Flight Booking Flow

Standards

WCAG 2.2 AA, Section 508

Platform

Windows 11, Chrome Browser

Goal

The goal of this audit was to evaluate how accessible and usable the MakeMyTrip flight booking experience is for users with visual, motor, cognitive, and assistive technology needs. The audit identifies barriers that may prevent users from completing key booking tasks and provides design recommendations to improve usability.

Audit Process

How the evaluation was conducted

01

Keyboard Review

Tested navigation, focus order, dropdown behavior, and interactive controls using keyboard-only interaction.

02

Screen Reader Review

Reviewed how links, icons, form fields, and page structure are announced to assistive technology users.

03

Visual Accessibility

Checked contrast, focus visibility, readability, and zoom behavior for low-vision users.

04

UX Recommendations

Translated accessibility issues into practical design improvements for a better user experience.

Tools Used

NVDA ANDI Axe DevTools WAVE Lighthouse Chrome DevTools Colour Contrast Analyser

Key Findings

Accessibility findings with image evidence

High WCAG 1.1.1 / 4.1.2

01. Social media icons lack accessible names

UX Observation: Social media icons rely only on visual recognition and do not provide a meaningful accessible name.

User Impact: Screen reader users may hear only “link” and cannot understand the destination or purpose.

Recommendation: Add descriptive labels such as “Follow us on Facebook” or “Follow us on Instagram.”

High WCAG 1.4.3

02. Helper text has insufficient contrast

UX Observation: Helper text for return date discounts appears in light gray on a white background.

User Impact: Users with low vision may struggle to read important booking guidance.

Recommendation: Increase contrast to at least 4.5:1 for normal text and define accessible color tokens.

Medium WCAG 1.4.10

03. Flight form does not reflow properly

UX Observation: At higher zoom levels, the booking form requires horizontal scrolling.

User Impact: Low-vision users who zoom the page must scroll horizontally to access form controls.

Recommendation: Use responsive layouts that stack booking fields vertically at smaller viewport widths.

High WCAG 2.1.1

04. Some controls are not fully keyboard accessible

UX Observation: Key interactive elements such as buttons and booking controls need complete keyboard support.

User Impact: Keyboard-only users may not be able to complete search or booking actions efficiently.

Recommendation: Ensure controls support Tab, Enter, Space, and Arrow key interactions using predictable patterns.

Medium WCAG 2.4.1

05. No skip link for repeated navigation

UX Observation: The page does not provide a visible-on-focus skip link to bypass repeated header content.

User Impact: Keyboard and screen reader users must repeatedly tab through navigation before reaching main content.

Recommendation: Add a “Skip to main content” link that appears when focused.

High WCAG 2.4.7

06. Focus states are difficult to identify

UX Observation: Interactive tabs and controls do not consistently show a strong visible focus indicator.

User Impact: Keyboard users may lose track of their current position on the page.

Recommendation: Use a consistent high-contrast focus ring across all interactive elements.

Medium WCAG 2.2.2

07. Auto-rotating carousel lacks user control

UX Observation: Promotional content rotates without clear pause, stop, previous, or next controls.

User Impact: Users may lose access to information before they finish reading or processing it.

Recommendation: Provide pause/stop controls and manual navigation for moving content.

High WCAG 1.3.1

08. Form labels and relationships need clarity

UX Observation: Form labels and relationships were reviewed to ensure they communicate field purpose clearly.

User Impact: Assistive technology users may struggle to understand what each booking field requires.

Recommendation: Ensure every input has a visible and programmatically associated label.

Design Opportunities

How the experience can be improved

Build accessibility-first form components.
Create consistent focus indicators across the design system.
Improve responsive layouts for zoom and small-screen users.
Provide user control for moving and animated content.
Include keyboard and screen reader checks in design handoff.

Outcome

Final Reflection

This audit highlighted how accessibility improvements directly improve the overall booking experience for all users. By addressing issues related to keyboard navigation, screen reader support, visual readability, responsive behavior, and form clarity, the product can become more inclusive, efficient, and easier to use.

Accessibility is not only a compliance requirement. It is a core part of user-centered design and helps create digital experiences that work for everyone.
